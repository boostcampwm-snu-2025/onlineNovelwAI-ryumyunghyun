# AI 활용 원칙 (AI Usage Principles)

## 1. 서비스 내 AI의 목적

AI는 온라인 소설 플랫폼에서 다음과 같은 역할을 수행합니다:

- **건설적인 피드백 제공**: 초보 작가들에게 구체적이고 실행 가능한 피드백 제공
- **다양한 독자 관점 시뮬레이션**: 서로 다른 페르소나를 통해 다각도의 의견 제시
- **스토리 컨텍스트 유지**: 여러 챕터에 걸친 스토리 일관성 파악 및 평가
- **객관적인 품질 지표 생성**: 정량화된 평가 기준으로 작가의 성장 추적

## 2. AI 모델 선택

### 선택된 모델: OpenAI GPT-4 (Primary) + Claude 3.5 Sonnet (Secondary)

**선택 이유:**

**GPT-4:**
- 창의적 글쓰기 피드백에 최적화
- 다양한 장르에 대한 폭넓은 이해
- 안정적인 API 제공
- 한국어 소설 분석 능력 우수

**Claude 3.5 Sonnet:**
- 긴 컨텍스트 윈도우 (200K tokens)로 장편 소설 분석 가능
- 섬세한 문학적 분석 능력
- 안전하고 윤리적인 응답 생성
- GPT-4 비교/보완용

### 비용 관리 전략

- 챕터당 리뷰 요청 제한: 일일 5회
- 짧은 챕터 (<1000자): GPT-3.5-turbo 사용
- 긴 챕터 (>5000자): Claude 사용
- 캐싱: 동일 챕터 재요청 시 이전 응답 재사용 (24시간)
- 토큰 제한: 응답당 최대 1500 tokens

## 3. AI 페르소나 설계 철학

### 핵심 원칙

1. **명확한 정체성**: 각 페르소나는 고유한 성격, 관심사, 평가 기준을 가짐
2. **일관성**: 동일 페르소나는 모든 리뷰에서 일관된 평가 스타일 유지
3. **건설적 균형**: 긍정적 피드백과 개선점을 균형있게 제시
4. **구체성**: 막연한 평가보다 구체적인 예시와 제안 제공

### 정의된 페르소나 (6종)

#### 1. 캐주얼 독자 (Casual Reader)
- **성격**: 평범한 독자, 재미와 몰입도 중시
- **평가 기준**: 흥미도, 가독성, 공감도
- **톤**: 친근하고 솔직함
- **예시**: "이 부분에서 주인공의 선택이 너무 갑작스러워서 이해가 안 됐어요."

#### 2. 문학 평론가 (Literary Critic)
- **성격**: 전문적이고 분석적, 문학적 가치 중시
- **평가 기준**: 문체, 주제 의식, 서사 구조, 상징성
- **톤**: 격식 있고 학술적
- **예시**: "반복되는 물의 이미지는 주인공의 심리적 변화를 효과적으로 상징합니다."

#### 3. 장르 팬 (Genre Enthusiast)
- **성격**: 특정 장르(판타지/로맨스 등)의 열렬한 팬
- **평가 기준**: 장르 문법 준수, 클리셰 활용, 독창성
- **톤**: 열정적이고 비교 분석적
- **예시**: "전형적인 환생물 설정이지만, 시스템 능력이 독특해서 신선해요!"

#### 4. 편집자 (Editor)
- **성격**: 실용적이고 꼼꼼한 전문가
- **평가 기준**: 맞춤법, 문법, 문장 구조, 가독성
- **톤**: 객관적이고 실무적
- **예시**: "3문단에서 시제가 혼용되고 있습니다. 과거형으로 통일하는 것이 좋겠습니다."

#### 5. 상업 출판인 (Commercial Publisher)
- **성격**: 시장성과 대중성 중시
- **평가 기준**: 트렌드 적합성, 상업적 잠재력, 타겟 독자층
- **톤**: 비즈니스적이고 전략적
- **예시**: "현재 웹소설 시장에서 20대 여성 독자들에게 어필할 수 있는 요소가 부족합니다."

#### 6. 창작 작가 동료 (Fellow Writer)
- **성격**: 공감적이고 격려하는 동료
- **평가 기준**: 창작 과정, 작가의 의도, 기술적 도전
- **톤**: 따뜻하고 지지적
- **예시**: "캐릭터의 내면 묘사에 많은 고민을 하신 게 느껴집니다. 잘하고 계세요!"

## 4. 윤리적 가이드라인

### 투명성
- 사용자에게 AI와 대화하고 있음을 명확히 고지
- AI 생성 리뷰임을 표시 (아이콘 또는 라벨)
- AI의 한계와 가능성에 대해 솔직하게 안내

### 인간 창작성 존중
- AI는 도구이며 인간 창작자를 대체하지 않음
- AI 제안은 참고 사항이며 강제가 아님
- 작가의 고유한 목소리와 스타일 존중

### 데이터 프라이버시
- 사용자 소설 콘텐츠는 AI 학습에 사용되지 않음
- 제3자와 콘텐츠 공유 금지
- 개인정보 보호법 준수

### 공정성과 편향 방지
- 특정 장르, 스타일, 주제에 대한 편견 배제
- 다양한 문화적 배경 존중
- 성별, 나이, 배경에 따른 차별 없음

### 유해 콘텐츠 필터링
- 폭력적, 선정적, 혐오적 콘텐츠 검토
- 부적절한 AI 응답 필터링
- 신고 시스템 운영

## 5. 컨텍스트 유지 전략

### 챕터 간 기억 메커니즘

**1. 대화 히스토리 저장**
```typescript
// 각 페르소나별로 챕터 리뷰 히스토리 유지
{
  personaId: "literary-critic",
  novelId: "novel-123",
  chapterHistory: [
    { chapterNumber: 1, summary: "주인공 소개, 마법학교 입학", rating: 8 },
    { chapterNumber: 2, summary: "첫 수업, 친구 관계 형성", rating: 7 }
  ]
}
```

**2. 임베딩 기반 시맨틱 검색**
- 이전 챕터들의 임베딩 벡터 저장
- 새 챕터 분석 시 관련 이전 내용 자동 검색
- Vector DB (Pinecone/Weaviate) 활용 고려

**3. 자동 요약 생성**
- 각 챕터 리뷰 후 AI가 핵심 내용 요약
- 누적 요약으로 긴 소설도 컨텍스트 유지
- 캐릭터, 플롯, 세계관 정보 구조화 저장

**4. 시스템 프롬프트에 컨텍스트 포함**
```
당신은 [페르소나명]입니다.
이 소설의 이전 챕터들을 읽었습니다:
- 챕터 1-3: [요약]
- 주요 캐릭터: [목록]
- 진행 중인 플롯: [목록]

이제 챕터 4를 평가해주세요. 이전 내용과의 일관성을 고려하세요.
```

## 6. 품질 관리

### AI 출력 검증
- 응답 길이 검증 (최소 200자, 최대 2000자)
- 평가 점수 범위 확인 (1-10)
- 필수 섹션 포함 확인 (장점, 개선점, 종합평가)

### 부적절한 응답 처리
- 욕설, 비하, 혐오 표현 필터링
- 지나치게 부정적이거나 모호한 응답 재생성
- 사용자 신고 시스템

### 반복성 감지
- 유사한 평가가 반복되는지 모니터링
- 템플릿화된 응답 방지
- 페르소나별 다양성 유지

### 사용자 피드백 수집
- "이 리뷰가 도움이 되었나요?" 버튼
- 피드백 데이터로 프롬프트 개선
- 주기적인 페르소나 업데이트

## 7. 개발 및 테스트 원칙

### 프롬프트 엔지니어링
- 명확하고 구조화된 프롬프트 작성
- Few-shot examples 제공
- 출력 형식 명시 (JSON 등)

### A/B 테스트
- 다양한 프롬프트 변형 테스트
- 사용자 만족도 측정
- 최적의 프롬프트 선택

### 모니터링
- API 응답 시간 추적
- 토큰 사용량 모니터링
- 오류율 추적
- 사용자 만족도 지표
