# Week 3 데모 개발 범위

## 데모 목표
3주차에 시연할 수 있는 최소 기능 제품(MVP) 개발

---

## 필수 개발 Feature

### 1. 사용자 인증 (User Authentication)
**우선순위**: 🔴 High

**기능**:
- 회원가입 (이메일, 비밀번호, 필명)
- 로그인/로그아웃
- JWT 토큰 기반 인증

**구현 범위**:
- 프론트: 회원가입/로그인 폼
- 백엔드: JWT 발급 및 검증 API
- 데이터베이스: User 테이블

---

### 2. 소설 프로젝트 관리 (Novel Management)
**우선순위**: 🔴 High

**기능**:
- 새 소설 프로젝트 생성
- 소설 목록 조회
- 소설 기본 정보 수정 (제목, 장르, 설명)

**구현 범위**:
- 프론트: 소설 생성 모달, 소설 카드 컴포넌트
- 백엔드: Novel CRUD API
- 데이터베이스: Novel 테이블

---

### 3. 챕터 작성 (Chapter Writing)
**우선순위**: 🔴 High

**기능**:
- 새 챕터 생성
- 챕터 내용 작성 (기본 텍스트 에디터)
- 자동 저장 (30초 간격)
- 단어 수 실시간 카운트
- 챕터 목록 사이드바

**구현 범위**:
- 프론트: 
  - 기본 textarea 에디터 (Rich Text는 추후)
  - 자동 저장 훅 (useAutoSave)
  - 챕터 네비게이션 컴포넌트
- 백엔드: Chapter CRUD API
- 데이터베이스: Chapter 테이블

---

### 4. AI 리뷰 시스템 (AI Review)
**우선순위**: 🔴 High

**기능**:
- 챕터에 대해 AI 리뷰 요청
- 2개 페르소나만 구현 (캐주얼 독자, 편집자)
- 리뷰 결과 표시 (평점, 텍스트, 장점/개선점)

**구현 범위**:
- 프론트:
  - 페르소나 선택 UI
  - 리뷰 생성 버튼
  - 리뷰 카드 컴포넌트
  - 로딩 상태 표시
- 백엔드:
  - OpenAI API 통합
  - 2개 페르소나 프롬프트 템플릿
  - Review 생성 API
- 데이터베이스: AIReview, AIPersona 테이블

**제약사항**:
- 컨텍스트 기억 기능은 **제외** (단일 챕터만 분석)
- 간단한 프롬프트만 사용
- 에러 처리 (API 실패 시)

---

### 5. 기본 대시보드 (Dashboard)
**우선순위**: 🟡 Medium

**기능**:
- 총 소설 수
- 총 챕터 수
- 총 단어 수
- 받은 리뷰 수
- 평균 AI 평점
- 최근 활동 (최근 작성한 챕터)

**구현 범위**:
- 프론트: 대시보드 레이아웃, 통계 카드
- 백엔드: 통계 계산 API
- 데이터베이스: 기존 테이블에서 집계

## 데모 시나리오

### 시연 흐름 (5-7분)

1. **회원가입 및 로그인** (30초)
   - 새 계정 생성
   - 필명 설정

2. **소설 프로젝트 생성** (30초)
   - "나의 첫 판타지 소설" 생성
   - 장르: 판타지
   - 간단한 줄거리 입력

3. **챕터 1 작성** (2분)
   - 챕터 제목: "이상한 숲에서 깨어나다"
   - 본문 작성 (500-1000자)
   - 자동 저장 기능 시연
   - 단어 수 카운트 확인

4. **AI 리뷰 요청** (2분)
   - "캐주얼 독자" 페르소나 선택
   - 리뷰 생성 버튼 클릭
   - 로딩 상태 (AI 응답 대기)
   - 리뷰 결과 확인:
     - 평점: 7/10
     - 장점: "흥미로운 시작..."
     - 개선점: "캐릭터 묘사 보강..."

5. **추가 리뷰** (1분)
   - "편집자" 페르소나 선택
   - 다른 관점의 리뷰 받기
   - 두 리뷰 비교

6. **대시보드 확인** (1분)
   - 통계 확인
   - 작성 현황
   - 평균 평점

## 성공 기준

### 기능적 요구사항
- ✅ 회원가입 후 로그인 가능
- ✅ 소설 생성 및 챕터 작성 가능
- ✅ 작성 중 자동 저장 동작
- ✅ AI로부터 리뷰 받기 성공
- ✅ 2개 페르소나 각각 다른 관점의 리뷰 제공
- ✅ 대시보드에서 기본 통계 확인 가능

### 기술적 요구사항
- ✅ API 응답 시간 5초 이내 (AI 제외)
- ✅ AI 리뷰 생성 60초 이내
- ✅ 모바일 반응형 UI
- ✅ 에러 처리 (네트워크 오류, API 실패)
- ✅ TypeScript 타입 에러 없음
- ✅ ESLint 경고 최소화

### 사용자 경험
- ✅ 직관적인 UI/UX
- ✅ 명확한 피드백 (로딩, 성공, 실패)
- ✅ 5분 내 회원가입 → 리뷰 받기까지 완료 가능
